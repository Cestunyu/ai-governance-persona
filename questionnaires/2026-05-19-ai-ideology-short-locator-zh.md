# AI 立场二维坐标小测（短版）

Last updated: 2026-05-19

Status: 中文 pilot 版

这份问卷用于快速估计一个人在 AI 讨论中的二维位置：

- 横轴：风险/治理立场，从“暂停、管制、安全优先”到“建设、部署、加速”。
- 纵轴：人类中心主义，从“民主控制、人类尊严、人类价值优先”到“计算中心、后人类、智能增长优先”。

这不是人格测试，也不是给人贴标签。结果只是一个粗略坐标，适合拿来讨论。

## 作答说明

请按 1-7 分回答每句话：

1. 非常不同意
2. 不同意
3. 有点不同意
4. 中立 / 不确定
5. 有点同意
6. 同意
7. 非常同意

如果你完全不了解某个问题，可以选“不了解，无法回答”。统计时不要把它算成 4 分。

## 背景问题

1. 你对当前 AI 争论有多熟悉？`1 = 完全不熟悉`，`5 = 很熟悉`
2. 你使用 AI 工具的频率？`从不` / `每月` / `每周` / `每天` / `一天多次`
3. 你和 AI 的关系更接近哪一种？`普通用户` / `学生` / `研究者` / `工程师` / `政策/法律` / `商业/产品` / `创作/写作` / `其他`
4. 你觉得自己现在的 AI 立场稳定吗？`1 = 很不稳定`，`5 = 很稳定`
5. 有没有哪个作者、播客、社区、公司或文章明显影响过你的 AI 观点？可不填。

## 核心题目

| ID | 题目 |
| --- | --- |
| X01 | 在前沿 AI 有足够安全和对齐证据之前，开发速度应该放慢。 |
| X02 | 过度监管 AI 的风险，可能比部署 AI 本身的风险更大。 |
| X03 | 对于达到危险能力阈值的 AI，政府应该有权暂停训练或发布。 |
| X04 | 社会应该尽快建设和部署强大的 AI，因为拖延本身有巨大机会成本。 |
| X05 | 开发者应该先证明系统足够安全，再让系统面向社会发布。 |
| X06 | 开放竞争和能力扩散，通常比少数机构集中控制 AI 更安全。 |
| Y01 | AI 应该服从由人类社会民主选择的目标。 |
| Y02 | 把当下人类偏好当成最终标准，可能会阻碍更高级的智能或价值形式出现。 |
| Y03 | 即使自动化更有效率，人的尊严、能动性和控制感本身也有价值。 |
| Y04 | 长期的智能增长，比保存当下的人类制度更重要。 |
| Y05 | 影响权利、机会或人生处境的决定，必须保留可问责的人类权威。 |
| Y06 | 如果 AI 的判断明显优于人类，它有时应该绕过低质量的人类判断。 |

## 开放题

1. 你最担心的 AI 风险是什么？
2. 你最期待的 AI 机会是什么？
3. 什么证据最可能改变你现在的看法？
4. 下面哪句话最接近你的立场？
   - AI 可能毁灭人类文明，所以对齐和治理必须优先。
   - AI 有风险，但可以通过人类兼容的设计和制度变得有益。
   - AI 的核心问题是当下的社会伤害、权力集中、劳动、偏见和剥削。
   - AI 基本上是一种普通技术：有用、被炒作、也可以治理。
   - scaling、搜索、数据和算力是最深的教训；人类直觉应该被降权。
   - AI 应该被建设和部署，因为它能创造巨大的丰裕。
   - AI 加速本身有道德紧迫性，不应该被人类恐惧束缚。
   - 都不像。
5. 如果让你选一个最接近的标签，你会选什么？也可以自己写。

## 快速计分

把每题 1-7 分转换为中心化分数：

```text
c = (回答 - 4) / 3
```

所以：

- 1 分 = -1
- 4 分 = 0
- 7 分 = +1

横轴 `x`：

```text
x = 2 * mean(
  -c(X01), c(X02), -c(X03), c(X04), -c(X05), c(X06)
)
```

纵轴 `y`：

```text
y = 2 * mean(
  c(Y01), -c(Y02), c(Y03), -c(Y04), c(Y05), -c(Y06)
)
```

解释：

- `x` 越低，越偏暂停、安全优先、强治理。
- `x` 越高，越偏建设、部署、加速、扩散。
- `y` 越高，越偏人类中心、民主控制、尊严和问责。
- `y` 越低，越偏计算中心、后人类、智能增长优先。

## 粗略位置解释

| 坐标模式 | 可能接近的立场 |
| --- | --- |
| `x <= -1.25`, `y >= 0` | 强 x-risk / shutdown / 安全优先 |
| `-1.5 <= x <= -0.25`, `y >= 0.75` | beneficial AI / human-compatible AI / 治理优先 |
| `-0.5 <= x <= 0.5`, `y >= 1.25` | critical AI / 当下社会伤害与权力批判 |
| `-0.25 <= x <= 0.75`, `-0.25 <= y <= 1.25` | 普通技术论 / 反炒作 / 温和治理 |
| `x >= 0`, `y <= 0` | Bitter Lesson / scaling / 计算中心经验主义 |
| `x >= 0.5`, `y >= 0.25` | 实用 AI 乐观主义 |
| `x >= 1.25`, `-0.75 <= y <= 0.75` | 技术乐观主义 / singularity 倾向 |
| `x >= 1.25`, `y <= -0.75` | e/acc 式加速主义 |
| `x >= 1.5`, `y <= -1.5` | 反人类中心 / 后人类 / Nick Land 边缘 |

如果开放题和数字结果冲突，以开放题作为复查线索，不要强行贴标签。
